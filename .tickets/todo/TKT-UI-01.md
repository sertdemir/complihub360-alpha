# TKT-UI-01: Build Wizard and Results Experience

## Description

The UI layer lacks the core engagement workflow required by NotebookLM documentation (Product Overview & System Architecture). The UI needs to be updated.

## Requirements

1. **Landing Page**: Implement Country selection, Search, and optional category selection.
2. **Wizard**: Implement a context sharpening wizard (4-5 steps). This is optional for users.
3. **Unified Results Page**:
   - Tabbed results view.
   - Provider list prominent on right sidebar.
4. **Primary CTA Modal**: Engagement request form capturing the structured intent from the user.
5. **Dashboard**: For registered users only.

## Expected Outcome

The `demo-app` or `vs1-demo` contains the screens matching the NotebookLM architecture and handles State Management for the Wizard flows.
