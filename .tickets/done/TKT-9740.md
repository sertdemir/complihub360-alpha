## 🏗️ Architectural Specification (AI Refined)

**Business Objective:**
Create the primary Dashboard Home screen for CompliHub360. The screen must feature a prominent, hero-style search interface in the upper half and a grid of functional modules/widgets in the lower half. The design will be generated using Google Stitch.

**Design Constraints (Neutral Foundation - Phase 0):**

1. **Palette:** Monochrome only (`bg-slate-950` canvas, `slate-900` surfaces, `slate-800` borders). No brand colors yet.
2. **Typography:** Inter font, utilizing `h1`, `h2`, `body`, and `small` hierarchy.
3. **Layout:** 12-column fluid grid, 4px baseline spacing (`gap-4`, `p-6` etc.).
4. **Resilience:** Fully responsive (`flex-col` on mobile, `grid` on desktop).

## ✅ Acceptance Criteria (QA-Sentinel)

- [ ] **Structural Layout:** Top half dedicated to "Hero Search" (Concept + extra input field). Bottom half displays functional "Modules" (Cards).
- [ ] **Stitch Generation:** The screen UI must be successfully generated and iterated using the `mcp_StitchMCP_generate_screen_from_text` tool.
- [ ] **Policy Verification:** The generated UI code adheres strictly to the `NEUTRAL_MODE_POLICY.md` (no hardcoded magical hex colors, borders restricted to structural dividers).
- [ ] **Responsiveness:** Search bar and module grid scale down gracefully to mobile viewports.
