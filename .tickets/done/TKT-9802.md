# React Board Epic Swimlanes & 'Waiting' Column

Refactor the `main.tsx` Kanban board to handle Epics and stalled agents.

## Requirements

1. Break down the monolithic `App` into modular components (`KanbanBoard`, `EpicSwimlane`, `TicketCard`).
2. Add a `Waiting` column between `Doing` and `Review`. Visual style should be yellow/orange to indicate a blocked state.
3. Group tickets returned by the API by their `epic` identifier. Render each Epic as a horizontal Swimlane (or distinct visual block). Non-epic tickets go into an "Unassigned" lane.

Rewrote `src/client/main.tsx` into modular components (`Board`, `EpicSwimlane`, `TicketCard`). The UI now groups tickets by their epic into distinct horizontal swimlanes and features a yellow/orange `Waiting` column.
